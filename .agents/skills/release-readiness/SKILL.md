# Release Readiness

Vérifier critères d’acceptation, tests, sécurité, migrations, sauvegarde/restauration, observabilité, réversibilité, décisions ouvertes et plan de retour arrière. Produire une décision Go/Go conditionnel/No-Go justifiée.
