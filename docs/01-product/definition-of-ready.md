# Definition of Ready

Une user story est prête lorsque :
- l’exigence source et la valeur métier sont identifiées ;
- l’acteur, le scénario nominal et les scénarios d’erreur sont décrits ;
- les règles métier sont stables ou les décisions ouvertes sont explicites ;
- les données, API, permissions, audit et impacts de migration sont identifiés ;
- les maquettes ou règles d’interface sont disponibles ;
- les critères d’acceptation sont testables ;
- les dépendances et données de test sont disponibles ;
- le niveau de sécurité et de confidentialité est classé.
