# Definition of Done

Une user story est terminée lorsque :
- le code est relu et conforme aux frontières d’architecture ;
- les tests unitaires, d’intégration, contractuels et pertinents E2E réussissent ;
- les permissions et règles SoD sont testées côté serveur ;
- l’audit et la corrélation sont vérifiés ;
- la migration Flyway et son test existent si nécessaire ;
- OpenAPI, modèle de données, documentation et traçabilité sont à jour ;
- aucune vulnérabilité critique ou secret n’est détecté ;
- les critères d’accessibilité et faible connectivité applicables sont vérifiés ;
- le Product Owner a accepté les critères de recette ;
- les preuves sont attachées à la pull request.
