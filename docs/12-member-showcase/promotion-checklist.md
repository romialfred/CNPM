# Checklist de promotion dans les contrats canoniques

Avant de commencer l’implémentation R4 :

- [ ] fermer ou cadrer `UX-DEC-003` à `UX-DEC-008` ;
- [ ] intégrer les chemins de `api-addendum.yaml` dans `docs/04-api/openapi.yaml` ;
- [ ] ajouter les permissions dans `docs/05-security/permissions.csv`, RBAC et SoD ;
- [ ] créer la prochaine migration Flyway et mettre à jour le dictionnaire de données ;
- [ ] ajouter les événements de publication et suspension à AsyncAPI ;
- [ ] ajouter le processus BPMN de modération/publication ;
- [ ] créer les stories et cas de recette dans les catalogues canoniques ;
- [ ] mettre à jour la matrice de traçabilité ;
- [ ] valider les actifs, droits, politiques de modération et URL ;
- [ ] exécuter revue architecture, données, API, sécurité, accessibilité et SEO.

Aucune copie parallèle du contrat ou des migrations ne doit subsister après promotion.
