# Rédaction des interfaces

## Principes

- Écrire pour l’action : verbe + objet.
- Employer un français simple, institutionnel et direct.
- Éviter les acronymes non expliqués lors du premier usage.
- Préférer « Enregistrer le brouillon » à « Sauvegarder » lorsque le statut compte.
- Préférer « Soumettre à validation » à « Envoyer » pour un workflow.
- Préférer « Aucun résultat » à « Rien trouvé ».
- Indiquer la conséquence d’une action dangereuse.

## Titres

- Page : nom de l’objet ou de la tâche, pas phrase marketing dans le back-office.
- Section : groupe de contenu spécifique.
- Dialogue : action à confirmer, par exemple « Annuler le reçu REC-2024-0052 ? ».

## Boutons

Bon : `Créer un membre`, `Rapprocher le paiement`, `Télécharger le reçu`, `Soumettre la vitrine`.

À éviter : `OK`, `Valider` sans objet, `Continuer` hors assistant, `Cliquez ici`.

## Messages de succès

Préciser résultat et prochaine étape :

> Le dossier d’adhésion a été soumis. Il sera examiné par le CNPM. Vous recevrez une notification lors de la décision.

## Messages d’erreur

Expliquer le problème et la récupération, sans code technique inutile :

> Le paiement n’a pas pu être rapproché, car le montant à affecter dépasse le solde disponible. Corrigez le montant puis réessayez.
