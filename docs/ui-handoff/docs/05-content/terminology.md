# Terminologie UI

| Terme recommandé | Définition / usage | À éviter |
|---|---|---|
| Membre | Organisation ou entité adhérente au CNPM | client, usager |
| Entreprise | Personne morale ou organisation enregistrée | membre lorsque le statut n’est pas acquis |
| Prospect | Entreprise non encore membre | nouveau membre avant validation |
| Cotisation | Obligation financière du membre | facture si juridiquement inexact |
| Appel de cotisation | Demande formelle pour une période | appel de paiement générique |
| Échéancier | Ensemble des tranches et dates | planning financier |
| Paiement reçu | Transaction captée, pas encore nécessairement confirmée | payé définitivement |
| Rapproché | Paiement associé à un membre et une obligation | confirmé si validation distincte |
| Confirmé | Paiement validé par le rôle habilité | encaissé si le sens est ambigu |
| Reçu officiel | Document émis au nom du CNPM après confirmation | reçu provisoire sans mention claire |
| Relance | Communication de recouvrement | menace, sommation sauf cadre juridique |
| Requête | Demande de service ou d’information | ticket dans l’interface membre |
| Réclamation | Contestation nécessitant traitement formel | requête si SLA différent |
| Groupement professionnel | Regroupement sectoriel du CNPM | groupe |
| Vitrine publique | Page Web promotionnelle du membre | mini-site autonome si gouvernance CNPM |
| Membre vérifié | Statut public selon critères CNPM | entreprise garantie / certifiée sans base |
| Brouillon | État modifiable non soumis | enregistré, si cela suggère une validation |
| En revue | Soumis à un validateur/modérateur | en attente sans contexte |
| Publié | Visible publiquement | validé si publication distincte |
| Suspendu | Accès ou publication temporairement bloqué | supprimé |

Les statuts doivent être définis dans un catalogue central et traduits de manière cohérente entre API, Web, mobile, exports et notifications.
