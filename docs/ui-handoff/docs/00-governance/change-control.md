# Gestion des changements UI/UX

## Versionnement

- **MAJEUR** : rupture de navigation, de structure ou de design system.
- **MINEUR** : ajout de composants, écrans ou variantes compatibles.
- **CORRECTIF** : correction de libellé, contraste, token ou comportement sans rupture.

## Processus

1. Ouvrir une proposition de changement contenant motif, captures, composants affectés et risque.
2. Vérifier l’impact fonctionnel, accessibilité, responsive, données et tests.
3. Obtenir validation Produit + UX ; ajouter Sécurité/Juridique si nécessaire.
4. Mettre à jour tokens, spécifications, stories et baselines dans la même pull request.
5. Ajouter une entrée au changelog.

Une modification visuelle isolée dans le code sans mise à jour du handoff est interdite.
