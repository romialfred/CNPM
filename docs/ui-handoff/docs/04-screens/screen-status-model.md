# Modèle de statut des écrans

- **Specifié** : route, gabarit et exigences fonctionnelles définis.
- **Référence visuelle** : une image guide la composition ; la fiche écran est normative.
- **À maqueter** : l’écran dérive d’un gabarit et de composants définis, sans PNG dédié.
- **Implémenté** : code intégré et Storybook disponible.
- **Validé UX** : revue responsive/accessibilité réussie.
- **Baseline visuelle** : captures approuvées et versionnées.

Le statut d’un écran est suivi dans la matrice XLSX et le backlog du projet, pas seulement dans ce document.
