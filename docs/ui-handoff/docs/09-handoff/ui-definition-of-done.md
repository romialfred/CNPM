# Definition of Done UI

Un écran ou composant n’est terminé que si :

- [ ] La source de vérité et la fiche ont été lues.
- [ ] Tous les composants proviennent du design system ou y ont été ajoutés.
- [ ] Les tokens sont utilisés sans valeur locale injustifiée.
- [ ] Les états nominal, chargement, vide, erreur et permissions sont couverts.
- [ ] Les données de démonstration sont déterministes.
- [ ] Le responsive est validé aux viewports applicables.
- [ ] Le clavier, le focus et axe passent.
- [ ] Les textes sont localisés et conformes au guide.
- [ ] Les tests unitaires/interactions passent.
- [ ] Les captures visuelles passent sous le seuil approuvé.
- [ ] La revue UX ne signale aucun écart bloquant.
- [ ] Les décisions nouvelles sont documentées.
- [ ] Les documents/Storybook/baselines sont mis à jour dans la même PR.
- [ ] Aucune donnée sensible, secret ou actif non autorisé n’est introduit.
