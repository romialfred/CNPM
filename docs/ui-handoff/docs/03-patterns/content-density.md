# Pattern — Densité et hiérarchie

## Back-office

- Les KPI sont limités aux décisions de la page.
- Les filtres occupent une ligne ou un panneau repliable, pas plusieurs cartes.
- Les tableaux dominent les pages de gestion.
- Les panneaux secondaires ne dépassent pas 30 % de la largeur quand la tâche principale est tabulaire.

## Public

- Sections aérées, largeur de lecture contrôlée.
- Les chiffres clés sont regroupés dans une bande légère.
- Les cartes de services ou projets contiennent un titre, un résumé bref et une action ; pas de longue prose.

## Portail membre

- Prioriser montant dû, échéance, action de paiement, demandes et documents.
- Limiter les KPI institutionnels qui ne concernent pas directement le membre.

## Mobile

- Une action principale par écran.
- Pas plus de cinq destinations principales.
- Les détails avancés passent dans des sections repliables.
