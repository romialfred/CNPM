# Pattern — Maître / détail

Utilisé pour paiements/rapprochement, requêtes et modération.

## Desktop

- Liste à gauche, détail au centre/droite.
- Sélection visible et conservée dans l’URL.
- Le détail ne se recharge pas entièrement lors d’une action locale.
- Les panneaux respectent une largeur minimale : liste 360 px, formulaire 420 px, preview 460 px.

## Tablette

- Liste puis drawer détail ; retour explicite.
- Les actions principales restent accessibles sans scroll horizontal.

## Mobile

- Navigation page-à-page.
- Le bouton Retour conserve filtres, scroll et sélection.

## États

Aucune sélection, chargement du détail, objet supprimé, accès retiré, modification concurrente et action en cours doivent être prévus.
