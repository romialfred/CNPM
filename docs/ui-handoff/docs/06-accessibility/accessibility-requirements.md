# Exigences d’accessibilité — WCAG 2.2 AA

## Cible

Conformité WCAG 2.2 niveau AA pour le Web public, le back-office et le portail membre. Les applications mobiles respectent les bonnes pratiques d’accessibilité Android/iOS et les critères équivalents.

## Exigences minimales

- structure de titres logique ;
- landmarks et lien d’évitement ;
- navigation complète au clavier ;
- focus visible et non masqué ;
- ordre de focus cohérent ;
- cibles d’au moins 24 × 24 CSS px, 44 × 44 recommandées sur tactile ;
- contraste texte et composants ;
- information non transmise par couleur seule ;
- labels, aides et erreurs programmatiquement associés ;
- zoom 200 % et reflow à 320 CSS px ;
- alternatives textuelles des images ;
- sous-titres/transcriptions pour médias ;
- tableaux structurés ;
- dialogs et drawers avec gestion du focus ;
- temps d’expiration annoncé et prolongeable selon sécurité ;
- mouvement réductible ;
- authentification sans épreuve cognitive non assistée ;
- statut accessible pour messages et chargements.

## Gate

Aucune page P0 ne peut être validée avec une violation axe critique/sérieuse ou un blocage clavier. Les contrôles manuels restent obligatoires, car l’automatisation ne couvre pas tous les critères.
