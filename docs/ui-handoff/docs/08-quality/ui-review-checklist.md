# Checklist de revue UI

## Fidélité

- [ ] Grille, alignements et dimensions respectés.
- [ ] Tokens utilisés ; aucune couleur/espacement magique.
- [ ] Hiérarchie typographique cohérente.
- [ ] Une seule action primaire par zone.
- [ ] Aucun cadre coloré décoratif ou ombre excessive.
- [ ] Données et libellés cohérents.

## États

- [ ] Chargement.
- [ ] Vide initial.
- [ ] Aucun résultat.
- [ ] Erreur récupérable.
- [ ] Accès interdit.
- [ ] Disabled/read-only.
- [ ] Confirmation/succès.
- [ ] Mobile hors connexion si concerné.

## Responsive

- [ ] 360, 390, 430.
- [ ] 768, 1024.
- [ ] 1280, 1440, 1672.
- [ ] Pas de scroll horizontal inattendu.
- [ ] Les tables se transforment correctement.
- [ ] Les actions restent visibles et ordonnées.

## Accessibilité

- [ ] Clavier/focus.
- [ ] Axe.
- [ ] Contraste.
- [ ] Zoom/reflow.
- [ ] Labels/erreurs.
- [ ] Titres/landmarks.
- [ ] Alternatives médias/graphiques.

## Produit

- [ ] Règles métier et RBAC respectés.
- [ ] Dates/montants/localisation.
- [ ] Télémétrie sans données sensibles.
- [ ] Audit sur actions sensibles.
