# Gates d’acceptation UI

| Gate | Exigence | Bloquant |
|---|---|---|
| Build | compilation, lint et types | Oui |
| Tests | unitaires et interactions | Oui |
| Storybook | stories P0 présentes | Oui |
| Accessibilité auto | 0 violation critique/sérieuse | Oui |
| Clavier | aucun blocage parcours P0 | Oui |
| Visual | seuil approuvé, aucun changement structurel inattendu | Oui |
| Responsive | viewports obligatoires | Oui |
| Performance | budgets de route respectés ou dérogation | Oui pour public P0 |
| Sécurité | pas de secret/XSS/permission seulement UI | Oui |
| UX | revue par paire ou UX | Oui |
| Produit | critères fonctionnels | Oui |

Une dérogation indique raison, risque, propriétaire, échéance et ticket de correction.
