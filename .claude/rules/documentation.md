# Documentation
- Utiliser le français pour les documents métier et l’anglais pour les identifiants techniques.
- Toute décision structurante crée ou met à jour un ADR.
- Toute exigence implémentée doit référencer son identifiant dans la story, le test et la pull request.
- Les hypothèses non validées sont marquées `À décider`, jamais présentées comme acquises.
