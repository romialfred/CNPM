# Implement User Story

1. Lire l’exigence, les règles métier et les décisions ouvertes.
2. Identifier API, données, permissions, audit, UI et tests.
3. Proposer un plan de fichiers et risques.
4. Implémenter par incréments.
5. Exécuter les contrôles applicables.
6. Mettre à jour la traçabilité et résumer les preuves.
